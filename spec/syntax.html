Velto Syntax

The Velto Markup Language is an XML‑based, declarative language. A Velto document describes UI structure and behaviour using a small set of custom tags and attributes. This page defines the core syntax rules that every Velto runtime must follow.

Document structure

A Velto document MUST contain exactly one <velto> root element. All other Velto elements MUST be descendants of this root. 

<velto>
  <content>
    <text1>Hello from Velto</text1>
  </content>
</velto>

Tag syntax

Velto tags follow XML syntax:

  • Tags MUST be properly opened and closed.
  • Tag names are case‑sensitive and MUST be lowercase.
  • Tags MAY be nested, subject to semantic rules defined in the Tags section.
<text1>Hello</text1>
<button>Click me</button>

Attribute syntax

Attributes MUST follow standard XML attribute syntax:

  • Attribute names MUST be lowercase.
  • Attribute values MUST be quoted with double quotes (").
  • Attributes MUST NOT appear more than once on the same element.
<button on-click:redirect="https://example.com">
  Go to example
</button>

Whitespace

Velto treats whitespace according to XML rules:

  • Whitespace between tags is preserved in text nodes.
  • Indentation and newlines are allowed and encouraged for readability.
  • Runtimes MAY normalise whitespace when mapping to DOM, consistent with HTML behaviour.

Comments

Velto supports XML comments. Comments MUST NOT affect runtime behaviour. 

<velto>
  <!-- This is a Velto comment -->
  <content>...</content>
</velto>

File extension

Velto documents SHOULD use the .velto file extension. Runtimes MAY support alternative extensions, but .velto is the canonical form.
