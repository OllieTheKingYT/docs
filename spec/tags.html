Velto Tags

Velto provides a minimal set of built‑in tags that map to HTML elements. Additional semantics are defined by attributes and behaviours. This section describes each standard Velto tag and its expected behaviour.

<velto>

The <velto> element is the root of every Velto document.

  • Required: YES (exactly one per document).
  • Allowed parents: none (it is the root).
  • Allowed children: any Velto element.
  • Default HTML mapping: <div>.

<content>

The <content> element represents the primary content container.

  • Required: NO.
  • Allowed parents: <velto> or other containers.
  • Allowed children: any Velto element.
  • Default HTML mapping: <div>.

<text1>

The <text1> element represents primary text content, typically paragraph‑level copy.

  • Allowed parents: any container tag.
  • Allowed children: text nodes only.
  • Default HTML mapping: <p>.

<text2>

The <text2> element represents secondary text content, such as captions or supporting text.

  • Allowed parents: any container tag.
  • Allowed children: text nodes only.
  • Default HTML mapping: <p> (runtime MAY style differently).

<button>

The <button> element represents an interactive control that can trigger actions or navigation.

  • Allowed parents: any container tag.
  • Allowed children: text nodes or inline elements.
  • Default HTML mapping: <button>.
<button on-click:redirect="https://example.com">
  Go to example
</button>

<image>

The <image> element represents visual content.

  • Allowed parents: any container tag.
  • Allowed children: none (self‑contained).
  • Required attributes: src.
  • Optional attributes: alt, behaviour attributes.
  • Default HTML mapping: <img>.
<image src="hero.png" alt="Hero image" />

Unknown tags

Unknown tags are not forbidden, but they are implementation‑defined. The reference Velto runtime maps unknown tags to <div> and emits a console warning. Future versions of the spec may define custom tag registration for user‑defined components.
